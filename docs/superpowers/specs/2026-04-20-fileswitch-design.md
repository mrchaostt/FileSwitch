# FileSwitch 设计文档

## 1. 项目概述

**项目名称:** FileSwitch
**项目类型:** 跨平台局域网文件传输工具
**核心功能:** 允许用户在同一局域网内的 Mac 和 Windows 电脑之间传输文件，支持扫描发现设备、文件多选、接收确认等特性。

**目标用户:** 在同一网络环境下需要在 Mac 和 Windows 电脑之间频繁传输文件的用户。

**最低系统要求:**
- Windows: Windows 7 及以上
- macOS: macOS 10.13 (High Sierra) 及以上

---

## 2. 技术方案

**技术栈:** Qt 6.x / C++

**选择理由:**
- Qt 提供原生跨平台支持，单一代码库支持 Windows 和 macOS
- 对 Windows 7 兼容性好
- Qt 网络模块成熟稳定，适合 UDP 广播发现和 TCP 文件传输
- 运行时性能好，内存占用低

---

## 3. 系统架构

### 3.1 双模式架构

软件采用"发送端/接收端"双模式设计，但使用同一个程序二进制文件：

| 模式 | 功能 | 网络行为 |
|------|------|----------|
| 发送端 | 扫描设备、选择文件、发送 | UDP 广播发现、TCP 连接发送 |
| 接收端 | 监听连接、确认接收、保存文件 | TCP 监听、UDP 广播响应 |

### 3.2 网络协议设计

**端口分配:**
- UDP 广播端口: 45678 (用于设备发现)
- TCP 传输端口: 45679 (用于文件传输和控制)

**发现机制:**
1. 发送端通过 UDP 广播发送"发现包"到 45678 端口
2. 所有在线的接收端收到广播后，回复自己的信息（主机名、IP、状态）
3. 发送端收集响应，列出可用的接收设备

**传输协议 (TCP):**
1. 发送端连接接收端的 45679 端口
2. 发送端发送传输请求包（包含发送者信息、文件列表）
3. 接收端弹出确认框，用户点击"接受"后建立连接
4. 文件数据分块传输（块大小: 64KB）
5. 传输完成后接收端保存文件到预设目录

### 3.3 数据结构

**设备信息 (UDP发现响应):**
```
{
    "hostname": "MacBook-Pro",
    "ip": "192.168.1.100",
    "status": "online"
}
```

**传输请求包:**
```
{
    "type": "transfer_request",
    "sender_ip": "192.168.1.101",
    "sender_hostname": "Desktop-PC",
    "files": [
        {"name": "photo.jpg", "size": 2048000},
        {"name": "document.pdf", "size": 512000}
    ]
}
```

**文件数据块:**
```
{
    "type": "file_data",
    "file_index": 0,
    "offset": 0,
    "data": <binary>
}
```

## 4. 功能模块

### 4.1 设备发现模块

| 功能 | 描述 |
|------|------|
| 扫描设备 | 点击"扫描"按钮，UDP广播发现局域网内接收端 |
| 显示本机IP | 状态栏显示本机局域网IPv4地址 |
| 设备列表 | 显示发现的所有设备（主机名 + IP） |
| 定时广播 | 接收端每隔5秒广播一次在线状态，发送端实时更新设备列表 |

### 4.2 文件选择模块

| 功能 | 描述 |
|------|------|
| 添加文件 | 打开文件选择对话框，支持多选 |
| 添加文件夹 | 选择文件夹，队列中显示文件夹内所有文件 |
| 文件队列 | 显示待发送的文件列表（名称、大小） |
| 移除文件 | 从队列中移除单个文件 |
| 清空队列 | 一键清空所有待发送文件 |

### 4.3 传输控制模块

| 功能 | 描述 |
|------|------|
| 发送请求 | 选择目标设备后，点击发送向接收端发起传输请求 |
| 接收确认 | 接收端弹窗显示发送者信息和文件列表，用户确认后开始传输 |
| 进度显示 | 显示当前传输进度条、传输速度、剩余时间 |
| 取消传输 | 发送或接收过程中可取消传输 |
| 传输完成 | 完成后显示成功提示，接收端自动保存文件 |

### 4.4 设置模块

| 功能 | 描述 |
|------|------|
| 下载目录 | 设置接收文件的默认保存目录 |
| 端口配置 | 显示/配置UDP和TCP端口（默认45678/45679） |

## 6. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 扫描无设备 | 显示"未发现任何设备，请确认接收端已启动" |
| 连接失败 | 显示"连接失败，请确认目标设备在线" |
| 传输中断 | 断开连接，显示"传输已中断"，清理临时文件 |
| 磁盘空间不足 | 传输前检查，提示"磁盘空间不足" |
| 文件名冲突 | 接收端自动重命名 (e.g., `photo.jpg` → `photo (1).jpg`) |

## 9. 关键实现细节

### 9.1 UDP广播发现实现
- 发送端: 向 `255.255.255.255:45678` 发送广播
- 接收端: 绑定 `0.0.0.0:45678` 接收广播并回复
- 接收端定时广播自己的在线状态 (每5秒)

### 9.2 TCP文件传输实现
- 使用 QTcpSocket / QTcpServer
- 文件分块传输，每块 64KB
- 大文件使用 QSaveFile 保证原子性写入

### 9.3 跨平台兼容性 (Windows 7)
- 使用 Qt 6.2+ (支持 Win7)
- 避免使用 Windows 10+ 特有API
- 网络字节序正确处理 (ntohl/ntohs/htonl/htons)
